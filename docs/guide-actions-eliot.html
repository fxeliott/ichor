Flip Win11 NSSM via script auto

Claude a déjà préparé le script scripts/windows/enable-cf-access-runner.ps1 qui fait le flip atomiquement.

  1. Ouvre PowerShell as Administrator :
    • Click menu Démarrer (logo Windows bas-gauche).
    • Tape powershell.
    • Click droit "Windows PowerShell" → Run as administrator.

§7 Rotate CF API token + SOPS re-encrypt — SÉCURITÉ

Découverte W95 — sécurité critique J'ai tenté d'automatiser §1 Tier 2-3 via Cloudflare API. Decrypt SOPS OK (clé age sur E:\age-key-ichor-2026-05-02.txt). MAIS : le token CF API stocké (cfut_*** ancien format) est RÉVOQUÉ par Cloudflare → "Invalid access token" sur tous endpoints. Le SOPS file infra/secrets/cloudflare.env contient un token mort.

Pourquoi rotater

  • Token mort : tout script qui l'utilise va échouer silencieusement.
  • Format cfut_ = ancien format CF (pré-2024). Probablement révoqué après migration CF API tokens.
  • Audit RUNBOOK-015 : rotation CF API tokens cadence 90 jours.

Étapes de rotation

  1. Va sur dash.cloudflare.com/profile/api-tokens.
  2. Si tu vois un token "ichor" old format cfut_ : click → Roll → confirm. Sinon click Create Token.
  3. Choisis "Custom token" et donne un nom : ichor-2026-05-10.
  4. Permissions :
    • Account · Cloudflare Tunnel · Edit
    • Account · Access: Apps and Policies · Edit
    • Account · Access: Service Tokens · Edit
    • Account · Account Settings · Read
    • Zone · Zone · Read
  5. Account Resources : Include All accounts (ou ton compte spécifique).
  6. Click Continue → Create Token. Copie le token immédiatement (affiché 1× seulement).
  7. Update SOPS :
export SOPS_AGE_KEY=$(cat 'E:/age-key-ichor-2026-05-02.txt')
sops --decrypt D:/Ichor/infra/secrets/cloudflare.env > /tmp/cf.env
# Edit /tmp/cf.env : remplace CLOUDFLARE_API_TOKEN=<NOUVEAU-TOKEN>
sops --encrypt --input-type dotenv --output-type dotenv /tmp/cf.env > D:/Ichor/infra/secrets/cloudflare.env
shred -u /tmp/cf.env

Validation

export SOPS_AGE_KEY=$(cat 'E:/age-key-ichor-2026-05-02.txt')
NEW_TOKEN=$(sops --decrypt D:/Ichor/infra/secrets/cloudflare.env | grep ^CLOUDFLARE_API_TOKEN= | cut -d= -f2-)
curl -s -H "Authorization: Bearer $NEW_TOKEN" https://api.cloudflare.com/client/v4/user/tokens/verify | \
  python -c "import json,sys; d=json.load(sys.stdin); print(d.get('result',{}).get('status','FAIL'))"

Doit retourner active.
