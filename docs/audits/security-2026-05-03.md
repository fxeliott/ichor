# Security audit — Ichor monorepo

**Date** : 2026-05-03
**Scope** : `apps/claude-runner`, `apps/api`, `apps/web`, `infra/`, `packages/`
**Mode** : read-only static review (no live exploitation)

## Executive summary

| Severity | Count |
|---|---|
| CRITICAL | 1 |
| HIGH     | 4 |
| MEDIUM   | 6 |

Defensive posture is generally strong : SQLAlchemy ORM (no string SQL),

## CRITICAL

### CRT-1 — Cloudflare Access not enforced on the production tunnel

- **Where** : `apps/claude-runner/src/ichor_claude_runner/config.py:44`,
  `apps/claude-runner/src/ichor_claude_runner/auth.py:54-56`,
  `infra/cloudflare/tunnel-config.yml`.
- **What** : `verify_cf_access` is correctly applied via
  `Depends(verify_cf_access)` on `/v1/briefing-task` and `/v1/usage`
  (`apps/claude-runner/src/ichor_claude_runner/main.py:128, 213`). However
  the live tunnel at `claude-runner.fxmilyapp.com` runs without
  Cloudflare Access policies — the entire JWT verification path is
  unused in production.
- **Risks** :
  1. If `ICHOR_RUNNER_REQUIRE_CF_ACCESS=false` ever flips, no auth at
     all : any internet client can POST `/v1/briefing-task` and drain
     Eliot's Max 20x weekly quota.
  2. Even with `require_cf_access=True`, since the edge does not enforce
     Access, a forged `Cf-Access-Jwt-Assertion` reaches the runner — the
     verify chain is correct, but see HIGH-1 (alg confusion in
     python-jose).
- **Status** : known + documented in
  [SECURITY.md → "Acknowledged residual risks"](../../SECURITY.md#acknowledged-residual-risks-phase-0).
  Phase-1 fix : enable Cloudflare Access (free for ≤ 50 users) +
  service-token JWT.
- **Mitigation today** :
  1. Tighten `lifespan()` to refuse start when
     `environment == "production"` and `require_cf_access == False`.
  2. Add a static shared-secret header verified before JWT (cheap,
     defense-in-depth).

## HIGH

### HIGH-1 — `python-jose` accepts the alg from the (untrusted) header

- **Where** : `apps/claude-runner/pyproject.toml:14` (`python-jose`
  unpinned upper bound), `apps/claude-runner/src/ichor_claude_runner/auth.py:80-96`.
- **CVE** : CVE-2024-33663 (algorithm confusion).
- **What** : `auth.py:92` passes
  `algorithms=[header.get("alg", "RS256")]` — the alg is read from the
  attacker-controlled header. A token with `alg: HS256` signed by the
  RS256 public key bytes used as HMAC secret would be accepted.
- **Fix** : pin `algorithms=["RS256"]` server-side (do not trust the
  header). Alternatively replace `python-jose` with `PyJWT >= 2.8`
  (actively maintained).
- **Status** : **FIXED** by commit applying RS256 pin (see
  [`apps/claude-runner/src/ichor_claude_runner/auth.py`](../../apps/claude-runner/src/ichor_claude_runner/auth.py)).

### HIGH-2 — WebSocket `/v1/ws/dashboard` no auth + no per-IP cap

- **Where** : `apps/api/src/ichor_api/routers/ws.py:22-29`,
  `apps/api/src/ichor_api/main.py:64`.
- **What** : `await websocket.accept()` is called unconditionally — no
  token, no origin check, no per-IP throttle. A malicious client can
  open thousands of WS, exhausting FDs (`LimitNOFILE=65536`).
- **Fix** : (1) Origin check against `cors_origins`. (2) Hard global
  connection cap. (3) Long-term : require auth.
- **Status** : **FIXED** — origin allow-list + global cap
  + per-connection logging added.

### HIGH-3 — All API REST endpoints unauthenticated, including write endpoints

- **Where** : `apps/api/src/ichor_api/routers/{briefings,alerts,bias_signals,news}.py`.
- **What** : `POST /v1/alerts/{alert_id}/acknowledge` is state-mutating
  with no auth. UUID enumeration is impractical, but design is wrong.
- **Fix** : behind Cloudflare Access (server-to-server) or session
  cookie + CSRF token.
- **Status** : **deferred to Phase 1** (Cloudflare Access enablement
  prerequisite, see CRT-1).

### HIGH-4 — `infra/secrets/.gitignore` referenced but missing

- **Where** : `infra/secrets/.sops.yaml:21-22` claims a defensive
  whitelist `.gitignore` that does not exist.
- **Fix** : create `infra/secrets/.gitignore` with explicit allow-list.
- **Status** : **FIXED** — `infra/secrets/.gitignore` added with
  block-everything-except-whitelist pattern.

## MEDIUM

### MED-1 — RSS collector parses untrusted XML with stdlib `xml.etree`

- **Where** : `apps/api/src/ichor_api/collectors/rss.py:29, 149`.
- **Risk** : billion-laughs / quadratic blowup (XXE not exploitable in
  Py3.12 stdlib defaults).
- **Fix** : `defusedxml.ElementTree.fromstring`.
- **Status** : **FIXED** — `defusedxml` dependency added,
  parser swapped.

### MED-2 — BBC RSS over plaintext HTTP

- **Where** : `apps/api/src/ichor_api/collectors/rss.py:71`.
- **Risk** : MITM injects feed body → prompt injection into Claude.
- **Fix** : change to `https://feeds.bbci.co.uk/...`.
- **Status** : **FIXED**.

### MED-3 — `seed_dev_data.py` uses f-string in `text("DELETE FROM ...")`

- **Where** : `apps/api/src/ichor_api/cli/seed_dev_data.py:57`.
- **Risk** : pattern is non-injectable today (hardcoded loop) but bad
  template.
- **Fix** : explicit `DELETE` statements + production environment guard.
- **Status** : **FIXED**.

### MED-4 — Persona file path mutable via env

- **Where** : `apps/claude-runner/src/ichor_claude_runner/config.py:37-38`.
- **Risk** : env-var override could point at credentials file.
- **Fix** : assert path is under known root at startup.
- **Status** : **FIXED** — startup assertion in `lifespan()`.

### MED-5 — CORS could be misconfigured to wildcard

- **Where** : `apps/api/src/ichor_api/main.py:52-58` + config.py.
- **Fix** : refuse `"*"` in production at startup.
- **Status** : **FIXED**.

### MED-6 — `claude-runner` `/healthz` fork-execs `claude --version` per call

- **Where** : `apps/claude-runner/src/ichor_claude_runner/main.py:85-114`.
- **Risk** : DoS via /healthz spam (with CRT-1 conditions).
- **Fix** : cache the version output for 60 s.
- **Status** : **deferred — claude-runner runs on Win11 user-mode and
  is reached only via authenticated tunnel; mitigated by CRT-1 fix**.

## Verified safe

- **No Anthropic API key in repo** — pattern search confirms ADR-009 /
  Voie D compliance.
- **No `dangerouslySetInnerHTML`, `innerHTML`, `eval`, `exec`, `pickle.load`,
  `yaml.load`, `os.system`, `shell=True`** anywhere outside dependencies.
- **SQLAlchemy 2.x parameterized queries** everywhere; only two `text()`
  uses (both hardcoded).
- **Pydantic strict input validation** : regex constraints + bounds on
  every Query parameter.
- **Subprocess execution** : `asyncio.create_subprocess_exec(*cmd_list)`
  with no shell, args validated by Pydantic Literal types.
- **CF Access JWT verify chain** : signature + audience + issuer +
  `require_exp` + `require_iat` all set. (HIGH-1 about alg trust now
  fixed.)
- **Rate limiter** : threading.Lock atomic, no window bypass.
- **SOPS+age multi-recipient** : Eliot + Hetzner age recipients both
  present in `.sops.yaml`. **No age private key in repo**.
- **Grafana Postgres password** : templated via Docker secret, no
  plaintext leak.
- **SSH/UFW/fail2ban** : hardened (no root pwd, no pwd auth,
  MaxAuthTries=3, fail2ban 24h).
